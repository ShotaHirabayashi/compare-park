## 2. 直接競合（「機械式駐車場×車種」領域）

### 2-1. velotaxi-sapporo.jp（現役整備士コータローの自動車ブログ）

| 項目 | 詳細 |
|------|------|
| **コンテンツ量** | 20記事以上 |
| **強み** | 整備士としてのE-E-A-T、車種別SEO対策が的確、ドメイン年数 |
| **弱み** | 読み物のみ（判定ツールなし）、駐車場データなし、網羅性に欠ける |
| **差別化ポイント** | トメピタは判定機能+駐車場データで圧倒的に実用的 |

### 2-2. car-repo.jp（最新自動車情報）

| 項目 | 詳細 |
|------|------|
| **コンテンツ量** | 3-5記事（年次更新） |
| **強み** | 「機械式駐車場に入るSUV」の年次更新で鮮度維持 |
| **弱み** | SUVのみ、個別車種ページ少ない |
| **差別化ポイント** | トメピタは全ボディタイプ204車種をカバー |

### 2-3. pdoctor.jp（ParkingDoctor+）

| 項目 | 詳細 |
|------|------|
| **コンテンツ量** | 30記事以上 |
| **強み** | メンテナンス会社運営でE-E-A-T最強、業界専門知識が深い |
| **弱み** | 車種マッチング機能なし、一般知識寄り |
| **差別化ポイント** | トメピタは車種×駐車場のマッチングで実用的UXを提供 |

## 3. 大手駐車場プラットフォーム（間接競合）

| サイト | 推定月間PV | サイズ検索 | 車種対応 | トメピタとの差異 |
|--------|-----------|-----------|---------|----------------|
| times-info.net (タイムズ) | 数千万 | サイズ制限表示あり | なし | 駐車場→検索のみ。逆引き不可 |
| repark.jp (リパーク) | 数千万 | 地図検索・料金計算 | なし | 予約・料金特化 |
| akippa.com | 数百万 | 予約型シェアリング | なし | 個人駐車場シェア |
| park-direct.jp | 数百万 | 月極検索+コラム | なし | 月極特化 |

**分析**: 大手はいずれも「駐車場→検索」の片方向。「車種を入力→適合駐車場を表示」するトメピタの逆引き型は市場に存在しない。

## 5. SWOT分析

### Strengths（強み）
- 車種×駐車場のマッチング機能は唯一無二
- 3段階判定（OK/ギリギリ/NG）の独自UX
- 204車種の包括的データベース
- エリア×車種4,692ページのロングテールSEO
- 充実した構造化データ（Car, ParkingFacility等）
- 高速な技術基盤（Next.js + Vercel + ISR）

### Weaknesses（弱み）
- コラム記事14本は競合対比で少ない
- ドメイン年数が短く、ドメインオーソリティが低い
- E-E-A-T信号が弱い（著者情報不足）
- 知識系コンテンツの欠如（重量制限、EV対応等）
- 駐車場データの網羅性がまだ不十分

### Opportunities（機会）
- 「[車種名] 機械式駐車場」で月間1万-3万検索の獲得余地
- EV普及でサイズ・重量問題の需要拡大
- 2026年4月の区分所有法改正で機械式駐車場への関心増
- AI検索での引用獲得（独自データが引用されやすい）
- 比較記事・ボディタイプ別まとめの空白領域

### Threats（脅威）
- 大手駐車場サイトがマッチング機能を追加する可能性
- 自動車メディア大手が駐車場コンテンツを拡充する可能性
- AI Overviewsでクリックが減少するリスク
- 薄いコンテンツ判定を受けるリスク（エリア×車種ページ）
