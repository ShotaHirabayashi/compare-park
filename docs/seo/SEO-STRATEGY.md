## 1. 現状分析サマリー

### サイト規模
| 指標 | 現状値 |
|------|--------|
| インデックス可能ページ | 約5,000ページ（静的8 + 車種204 + メーカー38 + エリア23 + エリア×車種4,692 + 駐車場N + サイズ別9 + 記事14） |

| ISR設定 | 24h〜7d（ページタイプ別） |

### 強み
- **独自ポジション**: 「車種→駐車場」の逆引きマッチングは競合不在
- **3段階判定**: OK/ギリギリ/NGの判定は他サイトにない独自価値
- **大規模ロングテールSEO**: エリア×車種の4,692ページ
- **豊富な構造化データ**: Car, ParkingFacility等の専門スキーマ実装
- **技術基盤**: Next.js + ISR + Vercelで高速・スケーラブル

### 弱み・改善点
- **コンテンツ量不足**: 14記事は競合対比で少ない（pdoctor.jp:30+, velotaxi-sapporo:20+）
- **知識系コンテンツの欠如**: 重量制限、EV対応、駐車場種類解説が未対応
- **ボディタイプ別まとめなし**: SUV/ミニバン等のカテゴリページが未作成
- **比較記事ゼロ**: compareカテゴリが空
- **E-E-A-T信号の弱さ**: 著者情報、専門性の提示が不足
- **「立体駐車場」キーワード未対応**: 同義語のカバー不足

## 5. GEO（Generative Engine Optimization）戦略

AI検索（Google AI Overviews、ChatGPT、Perplexity）での引用獲得を重視。

### トメピタが引用されやすいコンテンツ特性
- **一意のデータ**: 204車種×駐車場の適合判定データは他サイトにない
- **明確なテーブル形式**: 車種寸法と制限値の比較表
- **断定的な結論**: OK/NG/ギリギリの明確な判定結果
- **網羅性**: 全車種一覧は引用元として有用

### GEO施策
| 施策 | 詳細 |
|------|------|
| **llms.txt配置** | サイト概要、主要ページURL、データの説明を記載 |
| **引用しやすい文章構造** | 「〇〇は機械式駐車場（高さ1550mm制限）に✅入ります」のような明確な1文を各車種ページに配置 |
| **テーブルの最適化** | 車種寸法テーブルを統一フォーマットで全記事に配置 |
| **AI Crawlerのアクセス許可** | robots.txtでGPTBot, ClaudeBot, PerplexityBotを許可 |

## 6. 内部リンク戦略

### 現状の内部リンク構造（強み）
- パンくずリスト全ページ実装
- 車種→エリア（23区）×車種ページへのリンク
- 駐車場→同エリア他駐車場・対応車種
- 記事→carSlugで車種ページへリンク

### 強化ポイント
| リンク元 | リンク先 | 目的 |
|---------|---------|------|
| 車種ページ | 関連記事（cars/[slug]） | コンテンツ回遊・SEO |
| 記事ページ | 判定ツール（/check） | CVR向上 |
| 知識記事 | 車種ページ群 | 内部リンクジュースの分配 |
| compare記事 | 両車種ページ | 相互リンク |
| サイズ別ページ | 該当する車種記事 | トピッククラスター形成 |
| TOPページ | 人気記事・新着記事 | クロール促進 |

### トピッククラスター設計

```
[ピラーページ]                    [クラスター記事]

機械式駐車場ガイド(knowledge) ──→ 高さ制限解説
                              ├─→ 幅制限解説
                              ├─→ 重量制限解説
                              ├─→ 駐車場の種類
                              ├─→ EV×機械式駐車場
                              └─→ マンション駐車場

SUV×機械式駐車場(size-guide) ──→ ハリアー記事
                              ├─→ RAV4記事
                              ├─→ CX-5記事
                              └─→ フォレスター記事

ミニバン×機械式駐車場 ──────→ アルファード記事
                              ├─→ ヴォクシー記事
                              ├─→ セレナ記事
                              └─→ ステップワゴン記事
```

## 8. リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| **薄いコンテンツ判定** | エリア×車種4,692ページが低品質と見なされる可能性 | 各ページに独自の判定結果・コメントを含め、ユニークコンテンツ化 |
| **データの正確性** | 車種寸法データの誤りが信頼性を損なう | グーネット等の公式ソースとの定期照合、更新日の明示 |
| **AI Overviewsによるクリック減** | 回答がAI Overviewsで完結しクリックされない | 判定ツールへの誘導、詳細データの提供で「続きを見たい」動機を作る |
| **競合の追随** | 大手駐車場サイトが同様の機能を追加 | 車種データの深さ（グレード・オプション別）で差別化 |
